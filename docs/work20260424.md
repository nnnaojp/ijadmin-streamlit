## 3. init_raid_sequence() のリファクタリング (system_api.py)

### (1) os_disk パラメータを必須化

変更前: def init_raid_sequence(os_disk: str | None = None)
変更後: def init_raid_sequence(os_disk: str)

- 内部でのフォールバック検出(/proc/mounts 再解析)を廃止
- 呼び出し元(raid_config.py)で get_os_disk_info() の戻り値(1)を渡す設計に統一

### (2) sd_devices のデフォルト値をハードコードから動的検出に変更

- 変更前: sd_devices = ["/dev/sdb", "/dev/sdc", "/dev/sdd"]（固定）
- 変更後: /proc/mounts と glob を使い、未マウントかつOSディスクでない /dev/sdN を列挙
- /dev/sdb がOSディスクの場合は /dev/sda も RAID 対象に含まれる

### (3) sd_devices のバリデーション追加

- Step 2(wipefs)の直前で sd_devices が3台でない場合にエラー早期リターン

### (4) syslog ログ追加

- 各ステップ（関数開始、mdadm--stop、zero-superblock、wipefs、sgdisk、mdadm-C、mkfs）の開始時に write_syslog() を追加

## 4. アップデート画面 — アップデート本体処理の実装 (update.py)

ファイルアップロード → /tmp 書き出し後に以下を追加：

処理フロー:
  /tmp/<filename>.tgz 書き出し
  → tar tzf でトップディレクトリ名を取得
  → tar xzf /tmp/ に展開 → /tmp/<topdir>/
  → /tmp/<topdir>/update の存在確認
  → chmod 755 で実行権限付与
  → sudo ./update を cwd=/tmp/<topdir>/ で実行
  → rc=0 → 成功 / 非0 → エラー表示

エラーハンドリング対象:
  - tar tzf 失敗、tar xzf 失敗、update ファイルなし、スクリプト非0終了
